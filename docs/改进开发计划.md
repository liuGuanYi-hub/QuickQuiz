## 二、改进计划（分阶段）

### 第一阶段：核心闭环 ✅ 已完成（2026-04-25）

**目标：让练习流程跑通，错题本真实可用**

1. **练习模块完整化**
   - 完成答题 → 判分 → 结果展示全流程
   - 每次练习生成练习记录（exercise_records 表）
   - 答错题目自动进入错题本

2. **错题本功能连通**
   - 建立 `WrongQuestion` 实体，关联用户、题目、错题时间
   - 支持再次练习错题
   - 支持查看错题解析

3. **题目编辑**
   - 在题库管理页面增加编辑按钮和编辑 Modal

**数据模型变更：**
```sql
-- 练习记录表
CREATE TABLE exercise_records (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id BIGINT NOT NULL,
    question_count INT NOT NULL,
    correct_count INT NOT NULL,
    score DECIMAL(5,2),
    duration_seconds INT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 练习详情表（每道题的答题情况）
CREATE TABLE exercise_details (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    record_id BIGINT NOT NULL,
    question_id BIGINT NOT NULL,
    user_answer VARCHAR(500),
    is_correct BOOLEAN,
    FOREIGN KEY (record_id) REFERENCES exercise_records(id),
    FOREIGN KEY (question_id) REFERENCES questions(id)
);

-- 错题本表
CREATE TABLE wrong_questions (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id BIGINT NOT NULL,
    question_id BIGINT NOT NULL,
    wrong_count INT DEFAULT 1,
    last_wrong_at TIMESTAMP,
    mastered BOOLEAN DEFAULT FALSE,
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (question_id) REFERENCES questions(id)
);

-- 间隔重复复习计划表
CREATE TABLE spaced_repetition (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id BIGINT NOT NULL,
    question_id BIGINT NOT NULL,
    easiness_factor DECIMAL(3,2) DEFAULT 2.5,
    interval_days INT DEFAULT 1,
    repetitions INT DEFAULT 0,
    next_review_at DATE,
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (question_id) REFERENCES questions(id)
);
```

## 三、技术选型建议

| 功能 | 方案 | 备注 |
|------|------|------|
| AI 出题 | 硅基流动 / OpenAI / 通义 | 建议抽象 `AiProvider` 接口便于切换 |
| OCR | 百度文字识别 | 免费额度基本够用 |
| 间隔重复 | SM-2 算法 | 经典且成熟 |
| 缓存 | Redis | 可选，后期数据量大时引入 |
| 部署 | Docker | 便于分发和部署 |

---

## 四、注意事项

1. **数据库变更**：使用 Flyway 迁移脚本管理，禁止直接修改生产数据库结构
2. **敏感信息**：所有 API Key 均通过环境变量注入，不硬编码在配置文件中
3. **前后端协商**：AI/OCR 等功能需提前与前端确定接口格式和数据结构
4. **移动端优先**：练习和错题复习是高频移动场景，UI 适配不要拖到最后
